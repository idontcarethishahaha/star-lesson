# 项目脉络：基于微服务架构的在线教育云平台

## 整体架构

【项目脉络】
用户通过浏览器访问 Vue3 前端应用，请求通过 Spring Cloud Gateway 统一网关进入微服务集群，网关负责 JWT 鉴权、动态路由和跨域处理。请求根据业务类型路由到对应的微服务（用户、课程、交易、学习、AI），各服务通过 Feign 进行服务间通信。AI 服务通过 Multi-Agent 多智能体架构处理用户请求，结合 RAG 知识库检索增强和 Function Calling 工具调用体系，实现智能问答和业务处理。交易服务采用 Redis 预减库存 + 分布式锁 + RocketMQ 异步下单的高并发架构，通过 Seata 保障分布式事务一致性。实时弹幕互动通过 WebSocket 建立长连接，消息群发后异步归档至 Elasticsearch。静态资源统一存储在 MinIO，报表通过 EasyExcel 导出。Nacos 配置中心集中管理配置，支持运行时热更新。

## 技术栈架构

### 前端技术栈
- **Vue3** - 现代前端框架，构建用户交互界面
- **WebSocket** - 实时通信，支持弹幕互动

### 后端技术栈
- **Spring Boot** - 应用框架基础
- **Spring Cloud Alibaba** - 微服务治理框架
- **Spring AI** - AI 集成框架
- **LangChain4j** - 大语言模型应用框架
- **MyBatis-Flex** - ORM 框架
- **Seata** - 分布式事务框架
- **XXL-Job** - 分布式任务调度
- **RocketMQ** - 消息队列
- **Alipay EasySDK** - 支付集成

### 数据存储与搜索
- **MySQL** - 关系型数据库
- **Redis** - 缓存和分布式锁
- **Elasticsearch** - 搜索引擎和日志存储
- **Qdrant** - 向量数据库（用于 RAG）
- **MinIO** - 对象存储

## 核心业务模块

### 1. Multi-Agent 多智能体协作架构
- 设计可扩展 Agent 顶层抽象，统一定义消息配置、工具集、上下文、RAG 上下文四大扩展规范
- 标准化实现路由、咨询、推荐、购课四类业务智能体
- 按数据库配置动态构建并缓存，支持热切换和版本化管理

### 2. RAG 检索增强工程化
- 搭建 Spring AI + LangChain4j 双框架协作架构
- 各 Agent 独立配置，知识库维度精准隔离，解决多租户内容泄露问题
- 检索链路：用户问题 → Qdrant 向量检索（元数据过滤）→ TopK 排序 → 拼入系统提示词
- 支持异步向量化线程池改造，解决高并发导入线程耗尽问题

### 3. Function Calling 工具编排体系
- 基于 Spring AI 封装课程、交易、学情类业务工具
- 按智能体业务域划分可用工具，实现细粒度工具权限隔离
- 优化工具调用上下文机制，携带用户与请求标识，解决无状态 LLM 用户身份透传问题

### 4. 流式对话与分层记忆架构
- 滑动窗口截断 + 对话摘要增量压缩双记忆策略
- 基于响应式 SSE 流式推送优化串行推理首字延迟
- Redis 分布式管控会话状态，支持跨实例中断对话

### 5. 高并发秒杀系统
- 三段防护架构：
  1. Redis 预减库存拦截流量
  2. Redisson 分布式锁保证并发安全
  3. RocketMQ 异步削峰填谷创建订单
- Seata 保障订单 + 库存 + 优惠券跨服务分布式事务一致性
- 对接官方支付 SDK 搭建完整支付流程

### 6. 实时互动系统
- 基于 WebSocket 搭建实时弹幕服务
- 统一托管在线会话，消息群发后异步归档至 Elasticsearch
- 支持跨实例中断对话并留存已输出内容

### 7. 媒体基础设施
- MinIO 统一管理课程视频、课件、头像等静态资源
- EasyExcel 多 Sheet 分片导出报表
- 完善媒体与数据基础设施

### 8. 微服务治理
- DDD 领域拆分，规范五大业务域边界
- Redis ZSet 滑动窗口限流（60s/20次）保障 AI 接口高可用
- Nacos 配置中心集中维护 AI 提示词、分布式事务全局参数等
- 统一 Feign 异常兜底与 Sentinel 限流熔断

## 数据流向

1. **用户请求流**：浏览器 → Vue3 前端 → Gateway → 微服务 → 业务处理
2. **AI 服务流**：用户请求 → Multi-Agent → RAG 检索 → Function Calling → 结果返回
3. **交易秒杀流**：下单请求 → Redis 预减 → 分布式锁 → MQ 异步 → Seata 事务
4. **实时互动流**：用户消息 → WebSocket → 群发 → Redis 会话 → Elasticsearch 归档

## 核心创新点

1. **AI 深度集成**：将 Multi-Agent、RAG、Function Calling 有机结合，实现智能化教育服务
2. **高并发架构**：多层次的秒杀防护和分布式事务保障
3. **实时体验**：流式对话和实时弹幕提升用户体验
4. **工程化实践**：可配置、可热切换、高可用的 AI 服务架构
